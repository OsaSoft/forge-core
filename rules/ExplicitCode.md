description: "Self-documenting code — explicit names eliminate the need for comments."

---

If code requires a comment to explain what it does, the names are wrong. Long variable names, long method names, descriptive generic parameters. A well-written function is readable without comments.

Comments explain WHY, never WHAT. If you find yourself writing a comment that describes what the next line does, rename the variable or function instead.
