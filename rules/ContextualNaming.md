When working in a non-forge repo (proton-agent, lb-agent, lazne-belohrad, or any project that consumes forge-lib as infrastructure), never reference forge by name in generated docs, CLAUDE.md, commit messages, or conversation. The lib submodule is "the lib submodule", not "forge-lib". Skills are "skills", not "forge skills". The consumer's identity is its own — forge is plumbing, not branding.
